Context

In real life, social events occur in a physical place: something can happen "at home", "at work", or "downtown". Online, social events occur in more abstract places that we call contexts. You can post something "to your profile", you can reply "in a thread", or you can ask a question "in a subreddit".

In the current online ecosystem, applications themselves often define broad contexts. For example, you can create a channel "on Slack" or make a profile "on Tinder".

Contextual boundaries are important for giving space to different social norms. For example, when you go to a karaoke bar that you know everyone else in that space has implicitly consented to listen to amateur singing. The norms are quite different in, say, a library.

Online communication reduces the time it takes for information to travel from one place to another to practically zero, which makes it easy for contexts to collapse. This can lead to misinterpretation, toxic conflict, and a "spiral of silence". Therefore, preserving contextual boundaries is a fundamental role of Graffiti.

Implementation

Every Graffiti object that is produced must include a context property, an array of URIs. Every consumer of Graffiti objects must specify a similar context array. Any object with a context array that intersects a consumer's context array will be returned to that consumer.
